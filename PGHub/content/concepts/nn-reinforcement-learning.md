## intro
Supervised learning gives a model the right answer for every example. Reinforcement learning gives it almost nothing — just a stream of *rewards*, scalar feedback that says "that was good" or "that was bad," often long after the action that caused it. An **agent** acts in an **environment**: it observes a state, chooses an action, receives a reward and a new state, and repeats. Its goal is to learn a behavior — a **policy** — that maximizes the total reward collected over time, not just the immediate reward. This is how you train a system to play a game, balance a pole, route packets, or fine-tune a language model from human preferences: there is no labeled "correct move," only outcomes that are better or worse. The defining difficulties are that reward is delayed (the move that lost the game happened twenty turns ago) and that the agent must *try* actions to discover their value, all while the data it learns from is generated by its own changing behavior.

## whyItMatters
Reinforcement learning is the framework for learning to *act*, which makes it the natural fit for control, robotics, game-playing, recommendation, and sequential decision-making of every kind. It produced some of the most striking results in AI: TD-Gammon, Atari from raw pixels, AlphaGo and AlphaZero defeating world champions at Go and chess, and — most consequentially for current systems — reinforcement learning from human feedback (RLHF), the technique that aligns large language models with human preferences and turned raw text predictors into useful assistants. The reason RL deserves study even outside those headline applications is conceptual: it forces you to think about *credit assignment over time* (which earlier decision deserves blame or credit for a later outcome) and about the **exploration-exploitation tradeoff** — whether to repeat the best action you know or gamble on an untried one that might be better. These tensions appear far beyond RL, in A/B testing, clinical trials, and any system that must learn from the consequences of its own choices rather than from a fixed labeled dataset.

## intuition
Strip RL to its loop: an agent is in a **state** \(s\), picks an **action** \(a\), the environment returns a **reward** \(r\) and the next state \(s'\), and the loop repeats until the episode ends. The agent wants to maximize the **return** — the sum of rewards over the whole episode, with later rewards usually *discounted* by a factor \(\gamma<1\) so that sooner reward counts a little more and the sum stays finite.

The central trick for handling *delayed* reward is the **value function**. Instead of judging an action by its immediate reward alone, the agent learns the value of a state: the total future reward it expects to collect *starting from there*, if it behaves well afterward. A square one step from the goal is valuable even if standing on it pays nothing, because what follows is rich. Learning these values lets reward "flow backward" in time: a state inherits value from the good states it tends to lead to. This is the idea behind **temporal-difference learning** — you nudge your estimate of a state's value toward the reward you just got plus your estimate of the next state's value, \(V(s)\leftarrow V(s)+\alpha\,[\,r+\gamma V(s')-V(s)\,]\). The bracketed quantity, the **TD error**, is the surprise: how much better or worse things turned out than you predicted. Over many episodes, value seeps from the goal outward across the whole state space, building a map of where reward lives.

Now the deepest tension: to learn which actions are good, the agent has to *try* them, but to collect reward it should *use* the best action it has found. Always exploiting the current best (greedy) means you may never discover a better path you never tried; always exploring means you throw away reward on random flailing. This is the **exploration-exploitation tradeoff**. The simple, standard compromise is **\(\varepsilon\)-greedy**: with probability \(\varepsilon\) take a random action (explore), otherwise take the best known action (exploit), and **anneal** \(\varepsilon\) downward over time — explore boldly early when you know little, exploit confidently later once your value estimates have settled. Two broad families implement all this. **Value-based** methods (like Q-learning) learn the value of state-action pairs and act greedily with respect to them. **Policy-based** methods learn the action-choosing policy directly, adjusting it to make high-return actions more likely. Each has strengths, and **actor-critic** methods combine them — a policy (actor) chooses actions and a value function (critic) judges them — which is the basis of most modern deep RL.

## bruteForce
The naive way to find a good policy is **exhaustive search** or **pure random trial**: enumerate possible behaviors, run each, keep whichever collects the most reward. For a tiny problem with a handful of states and actions this can work, and random rollouts are a legitimate baseline. But it collapses immediately at scale. The number of possible policies grows astronomically with the number of states and actions, so enumeration is hopeless. Pure random play wastes essentially all of its experience: it does no **credit assignment**, so when a long episode finally ends in success it has no way to tell *which* of the hundred actions along the way actually mattered — it cannot propagate that outcome back to the decisions that produced it. It also ignores the structure that makes RL tractable: that states have *values* and that a good action in one state is informative about good actions in similar states. Random search treats every trajectory as an independent lottery ticket rather than as evidence that updates a reusable estimate. The value-function and policy-gradient methods exist precisely to reuse experience efficiently and to assign delayed credit, which brute force cannot do.

## optimal
The workhorse value-based method is **Q-learning**, which learns the action-value function \(Q(s,a)\): the expected discounted return from taking action \(a\) in state \(s\) and acting optimally thereafter. It is updated from each transition \((s,a,r,s')\) with a temporal-difference rule:
\[
Q(s,a)\;\leftarrow\;Q(s,a)+\alpha\Big[\,r+\gamma\,\max_{a'}Q(s',a')-Q(s,a)\,\Big].
\]
The target \(r+\gamma\max_{a'}Q(s',a')\) bootstraps off the agent's own current estimate of the best next value, and the bracket is the TD error driving the update; \(\alpha\) is the learning rate and \(\gamma\in[0,1)\) the discount. Q-learning is **off-policy** — it learns the optimal greedy values even while behaving \(\varepsilon\)-greedily to explore — and provably converges in the tabular case under standard conditions. When the state space is too large to tabulate (images, continuous control), replace the table with a neural network \(Q_\theta(s,a)\): this is **Deep Q-Networks (DQN)**, trained by minimizing the squared TD error, stabilized with an experience-replay buffer and a slowly-updated target network. The other family, **policy gradients**, parameterizes the policy \(\pi_\theta(a\mid s)\) directly and ascends the expected return; the REINFORCE estimator pushes up the log-probability of actions weighted by the return they led to, \(\nabla_\theta J=\mathbb{E}[\,\nabla_\theta\log\pi_\theta(a\mid s)\,G\,]\). **Actor-critic** methods combine both — the critic learns a value function to supply a low-variance baseline/advantage, and the actor updates the policy using it — and underlie modern algorithms like A2C, PPO, and the RLHF pipelines that fine-tune language models. Throughout, exploration is handled explicitly: \(\varepsilon\)-greedy or softmax sampling for value methods, stochastic policies for policy methods, annealed over training.

```python
import random

# Tabular Q-learning on a tiny gridworld; reward +1 at the goal, e-greedy exploration.
def q_learning(env, episodes=500, alpha=0.5, gamma=0.95, eps0=1.0):
    Q = {}                                              # Q[(state, action)] = value
    def q(s, a): return Q.get((s, a), 0.0)
    for ep in range(episodes):
        eps = max(0.05, eps0 * (1 - ep / episodes))     # anneal exploration down
        s = env.reset()
        done = False
        while not done:
            acts = env.actions(s)
            if random.random() < eps:                   # explore
                a = random.choice(acts)
            else:                                       # exploit the best known action
                a = max(acts, key=lambda x: q(s, x))
            s2, r, done = env.step(s, a)
            best_next = max((q(s2, x) for x in env.actions(s2)), default=0.0)
            td = r + gamma * best_next - q(s, a)        # temporal-difference error
            Q[(s, a)] = q(s, a) + alpha * td
            s = s2
    return Q
```

## complexity
time: O(episodes · steps_per_episode) update operations for tabular methods, each update being O(|A|) to take the max over actions; for deep RL each step additionally costs a network forward/backward pass. Convergence speed (episodes needed) depends on the discount, exploration schedule, and reward sparsity, not just on raw counts.
space: O(|S|·|A|) for a tabular Q-table — infeasible for large or continuous state spaces, which is exactly why function approximation (a neural net) is used, reducing storage to O(parameters) plus an O(buffer) replay memory.
notes: Sample efficiency, not asymptotic time, is RL's real bottleneck: the agent must generate its own data by acting, so the number of environment interactions needed dominates the cost. Reward sparsity and the exploration strategy drive that number more than any big-O term.

## pitfalls
- Confusing reward with return. Optimizing the immediate reward instead of the discounted sum of future rewards produces myopic, greedy behavior that walks into long-term traps. The fix: always optimize the return (with discount \(\gamma\)) via a value function, so delayed consequences are accounted for.
- Too little or too much exploration. A fixed greedy policy gets stuck in the first decent strategy it finds; constant random exploration never converges to good behavior. The fix: use \(\varepsilon\)-greedy (or a stochastic policy) and anneal exploration down over training — explore early, exploit late.
- A bad discount factor. Setting \(\gamma\) too low makes the agent ignore long-term reward; too close to 1 makes returns high-variance and slow to learn, and can diverge with function approximation. The fix: pick \(\gamma\) to match the horizon that matters for the task, commonly 0.9–0.99.
- The deadly triad in deep RL. Combining function approximation, bootstrapping, and off-policy updates (as DQN does) can diverge without stabilizers. The fix: use experience replay and a slowly-updated target network, and clip or normalize rewards.
- Reward hacking / misspecified reward. Agents optimize the reward you wrote, not the goal you intended, and will exploit any loophole. The fix: design and stress-test the reward signal, add shaping carefully (it can change the optimal policy), and watch for degenerate high-reward behaviors.

## interviewTips
- Define the loop crisply — state, action, reward, next state — then immediately separate immediate reward from the discounted return, and explain that the value function exists to handle *delayed* reward via credit assignment. Interviewers want to see you reason about consequences over time, not just one step.
- Contrast value-based and policy-based methods: value methods (Q-learning/DQN) learn what states/actions are worth and act greedily; policy methods (REINFORCE/policy gradient) tune the action distribution directly; actor-critic combines them. Mention off-policy versus on-policy and where each shines (discrete vs. continuous actions, sample efficiency vs. stability).
- Make the exploration-exploitation tradeoff concrete with \(\varepsilon\)-greedy and annealing, and connect RL to current practice — RLHF for language models, AlphaZero for games — to show you know why it matters beyond gridworlds.

## keyTakeaways
- RL learns from scalar reward, not labels: an agent observes a state, takes an action, and gets a reward plus next state, aiming to maximize the discounted return over an episode — the value function handles delayed reward by assigning credit across time.
- Value-based methods (Q-learning/DQN) learn the worth of states/actions and act greedily; policy-based methods adjust the policy directly; actor-critic combines a policy (actor) with a value estimate (critic) and underlies modern deep RL including RLHF.
- The exploration-exploitation tradeoff is unavoidable — \(\varepsilon\)-greedy with an annealed \(\varepsilon\) is the standard compromise — and the real cost in RL is sample efficiency (interactions needed), driven by reward sparsity and the exploration schedule.

## code.python
```python
import random

class GridWorld:
    # 1x4 corridor: states 0..3, goal at 3 gives +1. Actions: left/right.
    def reset(self): return 0
    def actions(self, s): return ['L', 'R']
    def step(self, s, a):
        s2 = max(0, s - 1) if a == 'L' else min(3, s + 1)
        return (s2, 1.0, True) if s2 == 3 else (s2, 0.0, False)

def q_learning(env, episodes=300, alpha=0.5, gamma=0.9):
    Q = {(s, a): 0.0 for s in range(4) for a in ['L', 'R']}
    for ep in range(episodes):
        eps = max(0.05, 1.0 - ep / episodes)        # anneal exploration
        s, done = env.reset(), False
        while not done:
            a = (random.choice(env.actions(s)) if random.random() < eps
                 else max(env.actions(s), key=lambda x: Q[(s, x)]))
            s2, r, done = env.step(s, a)
            best = max(Q[(s2, x)] for x in env.actions(s2))
            Q[(s, a)] += alpha * (r + gamma * best - Q[(s, a)])   # TD update
            s = s2
    return Q

Q = q_learning(GridWorld())
for s in range(4):
    print(f"state {s}: L={Q[(s,'L')]:.3f}  R={Q[(s,'R')]:.3f}")
```

```javascript
// Tabular Q-learning on a 1x4 corridor; goal at state 3 gives +1.
function step(s, a) {
  const s2 = a === 'L' ? Math.max(0, s - 1) : Math.min(3, s + 1);
  return s2 === 3 ? [s2, 1.0, true] : [s2, 0.0, false];
}
const actions = ['L', 'R'];

function qLearning(episodes = 300, alpha = 0.5, gamma = 0.9) {
  const Q = {};
  const q = (s, a) => Q[`${s},${a}`] ?? 0;
  for (let ep = 0; ep < episodes; ep++) {
    const eps = Math.max(0.05, 1 - ep / episodes);     // anneal exploration
    let s = 0, done = false;
    while (!done) {
      let a;
      if (Math.random() < eps) a = actions[Math.floor(Math.random() * 2)];
      else a = actions.reduce((b, x) => (q(s, x) > q(s, b) ? x : b), actions[0]);
      const [s2, r, d] = step(s, a); done = d;
      const best = Math.max(...actions.map(x => q(s2, x)));
      Q[`${s},${a}`] = q(s, a) + alpha * (r + gamma * best - q(s, a));  // TD update
      s = s2;
    }
  }
  return q;
}

const q = qLearning();
for (let s = 0; s < 4; s++)
  console.log(`state ${s}: L=${q(s, 'L').toFixed(3)}  R=${q(s, 'R').toFixed(3)}`);
```

## code.java
```java
import java.util.HashMap;
import java.util.Map;
import java.util.Random;

public class QLearning {
    static final String[] A = {"L", "R"};
    static int[] step(int s, String a) {
        int s2 = a.equals("L") ? Math.max(0, s - 1) : Math.min(3, s + 1);
        return s2 == 3 ? new int[]{s2, 1, 1} : new int[]{s2, 0, 0};   // [next, reward, done]
    }

    public static void main(String[] args) {
        Map<String, Double> Q = new HashMap<>();
        Random rng = new Random(0);
        double alpha = 0.5, gamma = 0.9;
        int episodes = 300;
        for (int ep = 0; ep < episodes; ep++) {
            double eps = Math.max(0.05, 1.0 - (double) ep / episodes);   // anneal
            int s = 0; boolean done = false;
            while (!done) {
                String a;
                if (rng.nextDouble() < eps) a = A[rng.nextInt(2)];
                else a = Q.getOrDefault(s + ",L", 0.0) >= Q.getOrDefault(s + ",R", 0.0) ? "L" : "R";
                int[] res = step(s, a); int s2 = res[0], r = res[1]; done = res[2] == 1;
                double best = Math.max(Q.getOrDefault(s2 + ",L", 0.0), Q.getOrDefault(s2 + ",R", 0.0));
                double cur = Q.getOrDefault(s + "," + a, 0.0);
                Q.put(s + "," + a, cur + alpha * (r + gamma * best - cur));  // TD update
                s = s2;
            }
        }
        for (int s = 0; s < 4; s++)
            System.out.printf("state %d: L=%.3f  R=%.3f%n", s,
                Q.getOrDefault(s + ",L", 0.0), Q.getOrDefault(s + ",R", 0.0));
    }
}
```

```cpp
#include <cstdio>
#include <map>
#include <string>
#include <random>
#include <algorithm>

struct Res { int s2, r, done; };
Res step(int s, char a) {
    int s2 = (a == 'L') ? std::max(0, s - 1) : std::min(3, s + 1);
    return s2 == 3 ? Res{s2, 1, 1} : Res{s2, 0, 0};
}

int main() {
    std::map<std::string, double> Q;
    auto key = [](int s, char a) { return std::to_string(s) + a; };
    std::mt19937 rng(0);
    std::uniform_real_distribution<double> uni(0, 1);
    double alpha = 0.5, gamma = 0.9; int episodes = 300;
    for (int ep = 0; ep < episodes; ep++) {
        double eps = std::max(0.05, 1.0 - (double)ep / episodes);   // anneal
        int s = 0, done = 0;
        while (!done) {
            char a;
            if (uni(rng) < eps) a = (uni(rng) < 0.5) ? 'L' : 'R';
            else a = Q[key(s, 'L')] >= Q[key(s, 'R')] ? 'L' : 'R';
            Res res = step(s, a); done = res.done;
            double best = std::max(Q[key(res.s2, 'L')], Q[key(res.s2, 'R')]);
            double cur = Q[key(s, a)];
            Q[key(s, a)] = cur + alpha * (res.r + gamma * best - cur);   // TD update
            s = res.s2;
        }
    }
    for (int s = 0; s < 4; s++)
        std::printf("state %d: L=%.3f  R=%.3f\n", s, Q[key(s, 'L')], Q[key(s, 'R')]);
    return 0;
}
```
